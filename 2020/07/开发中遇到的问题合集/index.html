开发中遇到的问题合集

23 七月 2020 发布在 tranquilpeak, features

​ 详情点我

1、es6写法模块化操作

使用es6写法实现模块化操作，在 nodejs 环境中不能直接运行，需要使用 babel 把 es6代码转换为es5代码，才可以在nodejs环境中运行

2、vue跨域问题

通过一个地址去访问另外一个地址，这个过程中，如果出现下面三个地方任意一个有不一样的，就会触发 跨域问题

① 访问协议（http、https） ② ip地址 ③ 端口号

解决办法：

①：在controller类上面加上 @CrossOrigin 注解

3、vue的created()只执行一次

路由多次跳转到同一个页面，页面的 created() 方法只会执行一次，后面不会在执行

解决方法：

使用vue监听



export default {
  data() {
    return {
        ......
       }
    }
  },
  created() { //页面进行渲染前执行
      ...
  },

 //此处为监听属性
  watch: {
     $route(to, from) { //当路由发生变化，就会执行该方法
        ... //调用自定义方法
     }
  },



  methods: {
        ...
    }
}

4、启动错误:

@org.springframework.web.bind.annotation.RequestParam java.util.List<?>. Not declared?

解决方法：

把请求参数 list 的泛型补充完整
